Introduction to GARCH Modeling¶

(Generalized) autoregressive conditionally heteroskedastic (GARCH) models were created for econometric and financial problems due to the variant volatility in financial data over time. The GARCH (p,q) model is given by

(1) $Y_{t} = \zeta + \delta y_{t-1} + \theta u_{t-1}^{2}$

Where $Y_{t}$ denotes equity returns for day $t$. This equation describes equity terms in terms of previous returns (the autoregressive term) and the error resulting from a combination of white noise as well as the portion of returns not explained by the autoregressive term. As a result, the error term $u_t$ is given by:

(2) $u_t = h_t e_t$ and $e_t$ ~ $N(0,1)$ and $I.I.D$ (a white noise process).

$h_t$ is the part of the error term that we can explain away due to trends in the financial data and it is given by:

(3) $h_t = \omega + \sum^{q}_{i=1} \alpha_i u_{t-1} + \sum^{p}_{i=1} \beta_i h_{t-1}$

In the model specification, q denotes the number of autoregressive $u_t$ terms and p denotes the number of autoregressive $h_t$ terms.

After the model is specified the parameters are learned using Maximum Likelihood Estimation (MLE) and the model is then evaluated based on the absense of autocorrelation in the residuals; if autocorrelation exists, then the model is ineffective in explaining all of the trends influencing equity returns.

This project’s goal was to predict equity returns based on macroeconomic shocks such as changes in Gross Domestic Product (GDP), unemployment rates, and consumer sentiment. By adding macroeonomic shock weights to the model, we can enhance the forecasting power of a GARCH(1,1) model for sector stock returns. The equity returns were modeled as a GARCH process. Equity returns $Y$ for stock $i$ at day $t$ are given by:

(4) $Y_{i,t} = E_{t-1}(Y_{i,t}) + \sum^{N}_{n=1} \beta ^{i}_{n} [F_{nt} - E_{t}(F_{nt})] + u_{i,t}$

Equation (4) shows equity returns explained by previous returns, information on macroeconomic shocks, and an error term.

(5) $E_{t-1}(Y_{i,t}) = Y^{i}_{0} + \delta^{i}_{1}Y_{i, t-1} + \delta^{i}_{ENDYR}ENDYR_{t} + \delta^{i}_{JAN}JAN_{t} + \delta^{i}_{MON}MON_{t} $

Equation (5) shows how the autoregressive term was modified to use $E_{t-1}(Y_{i,t})$ - this modification allows the model to account for portion of returns explained by macroeconomic effects, as well as financial effects such as spikes due to end-of-year, decreases due to the beginning of year, and data lag due to the Monday effect

Just as in the original GARCH(1,1) model, the error term is given by:

(6) $u_t = h_t e_t$ and $e_t$ ~ $N(0,1)$ and $I.I.D$

(7) $h_{i,t}^{2} = [h_{0}^{i} + \rho^{i}_{1} \frac{h_{i,t-1}^{2}}{\Gamma_{i,t-1}} + \theta^{i}_{1} u_{i, t-1}^{2}] * \Gamma_{i,t}$

(8) $\Gamma_{i,t} = exp[\sum^{N}_{n=1} \phi_{n} DF_{n,t}] $

Initial Statistics and Assumption Verification¶

Most standard statistical models assume the data is independent and identically distributed (IID) - the distribution doesn’t fluctuate and all items in the sample are taken from the same probability distribution. However, since equity returns are dependent on historical trends, initial statistical tests were done to check this assumption; both plots and the results of the Dickey-Fuller Tests are shown below.

As shown in the plots the mean and standard deviations are not indepependent and identically distributed. A more explicit stationary test can be done with the Dickey Fuller Test, which tests the null hypothesis of whether a unit root is present in an autoregressive model, or that the data is not IID and consequently the process is not stationary.

Since the p-value is greater than significance level, the null hypothesis cannot be rejected and therefore the process is non-stationary.

Model Training¶

The method below implements a Garch(1,1) model with added variables for macroeconomic shocks.

In [37]:
import statistics as st
import math
import numpy as np

def GARCH11_model(param, X, y):
    param = np.reshape(param, (len(param),-1))  
    omega, alpha, beta, a, b = param[0:5]
    shock_weights = param[5:]
    r = y - a - b * X[:, 0]  - shock_weights.T.dot(X[:, 1:].T)
    r = r[0]
    
    n = len(r)
    s = np.ones(n) * 0.01
    s[2] = st.variance(r[0:3])
    for i in range(3, n):
        s[i] = (omega + alpha * r[i - 1] ** 2 + beta * (s[i - 1]))
    logL = -((-np.log(s) - r ** 2 / s).sum())
    return logL

The data needs to be converted into a n x m matrix to feed into the methods above. The data also needs to be split into training and testing sets. Then the Maximum Likelihood Estimation can be run with the training data on the method above. In this case, X denotes the lagged returns, and microeconomic shocks and y denotes the equity returns

Model Evaluation¶

Once we have both models trained, we can assess goodness of fit, autocorrelation, and the predictive capacities the model has. The method below calculates the predicted return values using the previous day's returns, the model's estimated parameters, and macroeconomic shock data.

In [47]:
def fitted_values(est_params, X):
    est_params = np.reshape(est_params, (len(est_params),-1))  
    z = []
    estimated = []
    
    omega, alpha, beta, a, b = est_params[0:5]
    shock_weights = est_params[5:]
    estimated = a + b * X[:, 0] + shock_weights.T.dot(X[:, 1:].T)
    estimated = estimated[0]
        
    e = X[:, 0] - estimated
    n = len(e)
    h = np.ones(n) * 0.01
    h[2] = st.variance(e[0:3])
    for t in range(1, n):
        h[t] = (omega + alpha * e[t - 1] ** 2 + beta * (h[t - 1]))
    z = e * np.sqrt(h)
    return estimated + z

import statistics as st
import numpy as np


class Model:
    def __init__(self):
        self.estimated = []
        self.z = []

    def garch11_model(self, param, X):
        param = np.reshape(param, (len(param), -1))
        omega, alpha, beta, a, b = param[0:5]
        shock_weights = param[5:]
        r = X[:, 0] - a - b * X[:, 1] - shock_weights.T.dot(X[:, 2:].T)
        r = r[0]

        n = len(r)
        s = np.ones(n) * 0.01
        s[2] = st.variance(r[0:3])
        for i in range(3, n):
            s[i] = (omega + alpha * r[i - 1] ** 2 + beta * (s[i - 1]))
        logL = -((-np.log(s) - r ** 2 / s).sum())
        return logL

    def fitted_values(self, X, est_params):
        est_params = np.reshape(est_params, (len(est_params), -1))

        omega, alpha, beta, a, b = est_params[0:5]
        shock_weights = est_params[5:]
        self.estimated = a + b * X[:, 1] + shock_weights.T.dot(X[:, 2:].T)
        self.estimated = self.estimated[0]

        e = X[:, 0] - self.estimated
        n = len(e)
        h = np.ones(n) * 0.01
        h[2] = st.variance(e[0:3])
        for t in range(1, n):
            h[t] = (omega + alpha * e[t - 1] ** 2 + beta * (h[t - 1]))
        self.z = e * np.sqrt(h)
        return self.estimated + self.z
